System Requirements

Proof runs on macOS. The current universal build is intended for both Apple Silicon and Intel Macs.

Proof supports common photo formats including JPEG, PNG, TIFF, WebP, HEIF/HEIC, Fuji .HIF, and camera RAW formats such as RAF, CR2/CR3, NEF/NRW, ARW, DNG, ORF, RW2, and PEF.

Because Proof may be distributed outside the Mac App Store, macOS can show a first-open warning. Open the DMG, drag Proof into Applications, Control-click Proof, choose Open, then choose Open again in the warning dialog.

Catalogs

A catalog is Proof's local database for your library. It stores folder references, thumbnails, previews, ratings, flags, color labels, keywords, collections, and edit metadata.

Proof catalogs photos by reference. Your original files stay in their existing folders. Catalog work is separate from originals unless you use explicit file commands such as move, rename, export, or delete.

  • New Catalog: create a new .proof catalog.
  • Open Catalog: open an existing Proof catalog.
  • Catalog menu: switch catalogs, create another catalog, or reopen a recent catalog.

Adding and Syncing Folders

Click Add Folder to choose a folder of photos. Proof indexes the folder and its subfolders, reads metadata, and generates thumbnails in the background.

The left sidebar includes All Photographs, added root folders, expandable subfolders, and offline or missing-state indicators when source folders or files are unavailable.

Use Rescan when files change outside Proof. A rescan adds newly found photos, refreshes metadata for changed photos, marks missing files, and preserves ratings, flags, keywords, collections, and edit metadata.

Right-click a folder for actions such as Reveal in Finder, Rescan, Locate folder, Rebuild Folder Thumbnails, Remove Missing, and Remove from Catalog. Removing a folder from the catalog does not delete files from disk.

Filtering and Sorting

The filter bar narrows the current photo set. Available filters include rating comparison modes, picks, rejects, color label, keyword, capture date range, camera model, and lens.

When filters are active, the count shows matched photos out of the current scope. Sort options include capture time, filename, and rating. Use the direction button next to the sort menu to switch ascending or descending order.

Loupe, Keywords, and Survey

Loupe shows the current photo large with metadata, zoom controls, keywords, and a filmstrip. RAW and HEIF photos may take longer the first time they are opened because Proof decodes and caches a higher-quality preview.

Collections

Collections are manual groupings of photos. They do not move files and they do not duplicate photos on disk.

Use collections for selects, client shortlists, delivery candidates, portfolio sets, or temporary working groups. Create collections in the sidebar, add selected thumbnails through the context menu or drag and drop, then select a collection to view only its photos.

Deleting a collection removes only the grouping. Photos and files remain in the catalog and on disk.

Moving, Renaming, and Revealing Files

Proof is non-destructive by default, but some commands intentionally affect files on disk.

  • Reveal in Finder: show the original file or source folder.
  • Drag out of Proof: drag thumbnails to Finder, browsers, mail, or other apps using original file paths.
  • Move photos into folders: drag selected thumbnails onto a folder or subfolder in the sidebar. Proof asks for confirmation before moving originals.
  • Rename photos: press F2 or right-click and choose Rename. Patterns support {name}, {n}, and {n:3}.

Exporting and Deleting

Export writes new rendered files and does not modify originals. Select photos, choose Export, pick JPEG, PNG, TIFF, or WebP, and choose a destination file or folder.

For batch export, Proof writes one file per selected photo into the destination folder. Batch export can rename output files with the same {name}, {n}, and {n:3} pattern style used by batch rename.

Press Delete or Backspace to move selected photo files to the macOS Trash. Use Delete N Rejects after marking rejected photos with X. Missing-photo cleanup is separate and only removes dead catalog rows for files already gone from disk.

Preferences

Open Preferences with Command-, or the macOS app menu. Preferences include appearance, default export format, default thumbnail size, default sort, auto-advance, Trash confirmation, default catalog location, automatic rescan, and preview cache controls.

Preview Cache

Proof stores thumbnails, previews, and higher-quality develop/export sources in the catalog cache so browsing and Loupe review stay fast.

The Preview Cache section in Preferences shows cache location, total size, thumbnail size, preview size, develop/export source size, database size, and orphaned cache files.

Cache actions include setting a cache limit, using quick presets, cleaning orphaned files, rebuilding thumbnails, and clearing selected cache types. Clearing cache files does not remove photos, ratings, flags, keywords, or collections.

Troubleshooting

Thumbnails are blank or shimmering

Proof is still generating thumbnails. This is expected after adding a large folder or rebuilding cache.

A folder is offline

The source folder or drive is not currently available. Reconnect the drive, then right-click the folder and choose Locate folder if the path changed.

Photos show as missing

The files were moved, renamed, or deleted outside Proof. Run Rescan on the folder. If the files are gone permanently, use Remove Missing to clear the catalog rows.

RAW or HEIF files open slowly the first time

Proof is decoding and caching a higher-quality preview. Opening the same file again should be faster.

Filters hide photos unexpectedly

Check rating, pick/reject, label, keyword, date, camera, and lens filters in the filter bar. Clear active filters to return to the full current scope.

Export fails

Choose a writable destination and avoid exporting directly over an original source file. For batch export, choose a folder where Proof can create new files.

Cache is using too much disk space

Open Preferences, set a cache limit, clean orphaned files, or clear selected cache types. Proof regenerates cache files when needed.

Proof is designed around non-destructive catalog work. Keep normal backups of both your photo library and your Proof catalog. Proof is provided as-is, without warranty.
